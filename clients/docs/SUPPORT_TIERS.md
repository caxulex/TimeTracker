## Severity Definitions

### Critical (P1)
- Complete system outage
- Data loss or corruption
- Security breach
- All users unable to work

**Example:** TimeTracker is completely inaccessible, database corrupted

### High (P2)
- Major feature not working
- Significant performance degradation
- Workaround exists but impacts productivity

**Example:** Timer not saving entries, reports not generating

### Medium (P3)
- Minor feature issue
- Cosmetic problems
- Non-critical functionality affected

**Example:** Date format displaying incorrectly, export missing a column

### Low (P4)
- Questions about usage
- Feature requests
- Documentation clarification
- Enhancement suggestions

**Example:** "How do I export to Excel?", "Can you add dark mode?"

## Excluded from Support

The following are NOT covered under standard support:

1. **Third-party integrations** not officially supported
2. **Custom code modifications** made by client
3. **Training** beyond initial onboarding
4. **Data migration** from other systems
5. **Infrastructure issues** on client-managed servers
6. **Issues caused by** unauthorized modifications

These services are available as paid add-ons.

---

## Support Best Practices for Clients

To get the fastest resolution:

1. **Check documentation first** - Many answers are there
2. **Include details** - Error messages, screenshots, steps to reproduce
3. **Use correct severity** - Helps us prioritize correctly
4. **One issue per ticket** - Easier to track and resolve
5. **Respond promptly** - Keeps issues moving forward
